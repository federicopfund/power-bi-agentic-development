# v26.24.2

Strengthens the Desktop bridge verification loop in `pbir-cli` (reports plugin)
and fixes a contradiction in `connect-pbid` (pbi-desktop plugin):

- pbir-cli SKILL.md: new Critical Rule 6 requires `pbir desktop refresh` after
  every change when the report is open in Desktop (unless the user opts out),
  followed by `pbir desktop screenshot` and PNG inspection; for multi-change
  requests the agent asks up front whether to refresh per step or once at the
  end. The Validation section now states a change is not complete until it has
  been seen rendering; the formatting workflow example ends with the
  refresh/screenshot loop
- connect-pbid desktop-bridge.md: the raw named-pipe path is now explicitly
  fallback-only. When the `pbir` CLI is installed, `pbir desktop
  list/refresh/screenshot` must be used instead of PowerShell against the
  pipe. Previously the reference said "no wrapper CLI", contradicting the
  skill's own preference for `pbir`
- Versions: reports and pbi-desktop plugins, pbir-cli skill, and marketplace
  metadata bumped to 26.24.2
