# v26.24.3

`pbir-cli` (reports plugin): graceful degradation when the Desktop bridge is
unavailable, complementing the verify-loop mandate from 26.24.2:

- Critical Rule 6 now requires checking availability once with
  `pbir desktop list` before starting the refresh loop, instead of retrying
  the bridge after every change when it cannot work
- New "When the bridge is unavailable" note in Desktop Integration:
  distinguishes the preview feature being off (relay the enable steps once,
  ask the user whether to turn it on) from no instance having the report
  open; falls back to `pbir validate --all` and the permission-gated
  publish-to-sandbox path until the bridge works or when not on Windows
- Versions: reports plugin, pbir-cli skill, and marketplace metadata bumped
  to 26.24.3
